# 티셔츠 관리 시스템 — 로컬 실행 안내 (Windows, 비개발자용)

이 문서는 **다른 컴퓨터에서 하던 프로젝트**를 **이 PC에서 처음 실행**할 때 따라 하면 됩니다.

---

## ✅ 이미 완료된 작업 (이번에 설정된 것)

- **가상환경** `venv` 생성
- **패키지 설치** (`requirements.txt` 기준)
- **환경 설정 파일** `.env` 생성 (로컬용 최소 설정)
- **데이터베이스** 마이그레이션 완료 (SQLite 사용)
- **관리자 계정** 생성됨 (아이디: `admin`, 비밀번호: `admin1234`)

---

## 🚀 매번 이 PC에서 실행할 때 (3단계)

### 1단계: 프로젝트 폴더로 이동

파일 탐색기에서 아래 폴더를 엽니다.

```
c:\Users\wonki\tshirt-management
```

**PowerShell** 또는 **명령 프롬프트**를 쓰려면:

```powershell
cd c:\Users\wonki\tshirt-management
```

---

### 2단계: 가상환경 켜기

**PowerShell**에서:

```powershell
.\venv\Scripts\Activate.ps1
```

맨 앞에 `(venv)` 같은 표시가 나오면 성공입니다.

**명령 프롬프트(cmd)** 에서는:

```cmd
venv\Scripts\activate.bat
```

---

### 3단계: 개발 서버 실행

가상환경이 켜진 상태에서:

```powershell
python manage.py runserver
```

아래처럼 나오면 정상입니다.

```
Starting development server at http://127.0.0.1:8000/
Quit the server with CTRL+BREAK.
```

## ❓ 자주 하는 질문

### "이 PC를 처음 켰는데 다시 실행하려면?"

매번 **2단계(가상환경 켜기)** → **3단계(runserver)** 만 반복하면 됩니다.  
`venv`와 `.env`, DB는 이미 있으므로 **1단계만** 프로젝트 폴더로 이동하면 됩니다.

### "다른 컴퓨터에서 코드만 가져왔을 때 (venv 없을 때)"

아래를 **처음 한 번만** 순서대로 실행합니다.

1. **Python 설치 확인**  
   `python --version` → 3.10 이상이면 OK.

2. **가상환경 생성**  
   ```powershell
   cd c:\Users\wonki\tshirt-management
   python -m venv venv
   ```

3. **가상환경 켜기**  
   ```powershell
   .\venv\Scripts\Activate.ps1
   ```

4. **패키지 설치**  
   ```powershell
   pip install -r requirements.txt
   ```

5. **환경 설정**  
   `.env` 파일이 없다면, 프로젝트 폴더에 `.env.example`을 복사해 `.env`로 이름을 바꾼 뒤,  
   최소한 아래만 넣어 두면 됩니다.  
   ```
   SECRET_KEY=django-insecure-local-dev-key-change-in-production
   DEBUG=True
   ALLOWED_HOSTS=localhost,127.0.0.1
   ```

6. **DB 마이그레이션**  
   ```powershell
   python manage.py migrate
   ```

7. **관리자 계정** (없을 때만)  
   ```powershell
   python manage.py create_admin
   ```  
   (비밀번호는 `admin1234`로 생성됩니다.)

8. **서버 실행**  
   ```powershell
   python manage.py runserver
   ```

### "테스트 데이터가 필요해요"

```powershell
.\venv\Scripts\Activate.ps1
python manage.py generate_test_data --clear
```

기존 데이터를 지우고 테스트용 주문/제품 데이터를 만듭니다.

### "네이버 스마트스토어·구글 드라이브·Cloudinary는?"

지금은 **설정하지 않아도** 로컬에서 실행 가능합니다.  
나중에 연동할 때 `.env` 파일에 해당 API 키/설정을 추가하면 됩니다.  
자세한 항목은 `.env.example`을 참고하세요.

## 📁 프로젝트 구조 (참고)

- `manage.py` — Django 관리/실행 진입점
- `requirements.txt` — 필요한 Python 패키지 목록
- `.env` — 로컬 설정 (비밀키, DB 등). **git에 올리지 마세요.**
- `venv/` — 가상환경 폴더 (이 폴더만 있으면 2·3단계만 반복하면 됨)
- `db.sqlite3` — 로컬 DB 파일 (데이터가 여기 저장됨)

더 자세한 기능 설명은 **README.md**를 보시면 됩니다.
